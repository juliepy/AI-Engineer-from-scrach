# AI Agent Memory 系统设计

> 来源：Shawn 视频文稿（Working Memory + 三柱记忆 + 门控巩固）

---

## 一、核心结论（先记住这句）

**现代 Agent Memory = 工作内存（上下文）+ 三类长期记忆（程序 / 语义 / 情节）+ 门控巩固。**  
会话本身是短暂的；真正跨会话、省 Token 的能力，来自「按需召回」和「把情节摘要成耐久事实」。

---

## 二、为什么需要 Memory

ChatGPT / Claude 能记住「你是谁、上周聊过什么、你的公司是什么」，并不是模型本身有状态，而是背后挂了一套 Memory 系统。

对 AI 产品 / Agent 来说：

- 只靠当前对话历史 → 会话结束就丢光
- 全量历史塞进 Prompt → 贵、慢、不准（上下文窗口有上限）
- 需要：**选择性注入上下文 + 持久化存储 + 定期压缩成核心事实**

---

## 三、整体架构

```
用户 Prompt
    │
    ▼
┌─────────────────────────────────────┐
│  Working Memory / Context RAM       │
│  · System Prompt（角色）            │
│  · Current Chat History（本会话）   │
│  · User Prompt（本轮问题）          │
│  · 从三类记忆召回的片段             │
└─────────────────┬───────────────────┘
                  │
                  ▼
            LLM（Q&A Agent）
                  │
                  ▼
               回复用户
                  │
                  ▼
         写入 Episodic Memory（活动日志）
                  │
         Gate（满 N 轮 / N 次活动）
                  │
                  ▼
           Summarizer Agent
                  │
                  ▼
         Steal into Facts → Semantic Memory
```

会话（AI Agent Session）是 **ephemeral（短暂）** 的：不主动落库，对话结束就没了。  
Working Memory 只是「这一次 LLM 调用能看见的上下文」，不是长期记忆。

---

## 四、Working Memory（工作内存 / Context RAM）

### 是什么

送进 LLM 之前组装好的**当前上下文**，相当于 Agent 的「内存条」。

### 基础三件套

| 组件 | 作用 | 例子 |
|------|------|------|
| **User Prompt** | 本轮用户问题 | 「今天天气怎么样？」「怎么理解相对论？」 |
| **Chat History** | 本会话已发生的对话 | 电商客服这一通会话里的前后文 |
| **System Prompt** | 角色 / 行为约束 | 「你是 Elon Musk，用他的口吻回答」 |

没有额外记忆时，Agent 只知道：系统角色 + 本会话历史 + 当前问题。  
它不知道：产品库存、历史订单、客户偏好、过去投诉——这些都不在当前会话里。

### 局限

- 会话级：跨会话不保留
- 容量有限：不能把公司十年文档全塞进去
- 需要「三柱记忆」把更丰富、更省 Token 的信息注入 Working Memory

---

## 五、三柱记忆（补全 Working Memory）

| 类型 | 存什么 | 存储形态 | 注入方式 |
|------|--------|----------|----------|
| **Procedural Memory** | 怎么做、怎么表现 | 文件（如 `skill.md`） | 作为技能 / 习惯读入上下文 |
| **Semantic Memory** | 耐久事实、画像、品牌 | 向量库 | RAG Top-K 召回 |
| **Episodic Memory** | 带时间戳的事件 / 活动日志 | 向量库 | RAG Top-K 召回 |

### 1. Procedural Memory（程序记忆）

**本质**：教 Agent「遇到某种情况该怎么做」——习惯 / 流程 / Skill，不是「记得某件事」。

- 例：客户很生气 → 礼貌回应并道歉
- 类比：教员工 / 小孩的行为规范
- 落地：Markdown 技能文件（`skill.md`）等，写入 Working Memory

### 2. Semantic Memory（语义记忆）

**本质**：相对稳定、可复用的**耐久事实**。

存什么：

- 公司 / 产品是什么、卖什么、品牌怎么对外
- 用户画像（常联系的客户是谁、偏好是什么）
- 不常变的业务事实

怎么用：

- 存向量库 → **RAG（Retrieval Augmented Generation）**
- Top-K 相似度检索，只取最相关片段注入 Working Memory
- 为什么必须「选择性拉取」：十年公司资料巨大，全塞进 LLM 既贵又不可行；上下文过大还会变慢、变糊

> 基础模型不知道你的小公司是谁时，必须自己把 durable facts 存库，再靠 RAG 按需取。

### 3. Episodic Memory（情节记忆）

**本质**：带时间线的**事件 / 活动记录**，不是静态事实。

存什么：

- 谁在哪天买了什么、何时发货、上次投诉是什么时候
- 本会话结束后落库的 chat history / 活动日志

和 Semantic 的共同点：都走向量库 + RAG Top-K。  
和 Semantic 的区别：Episodic 是「发生过什么」（dated events），Semantic 是「长期成立的事实」。

写入路径：Agent 回复后 → 把消息 / 活动写回 Episodic → 日志持续增长。

## 六、门控巩固：Episodic → Semantic（Steal into Facts）

### 问题

若每次都从巨大的 Episodic 里搜「关于你 / 你公司的一切」：

- 库太大、贵、慢
- ChatGPT / Claude 的「可编辑 Memory」往往不长，却会随对话更新——靠的是**摘要后的耐久事实**，不是全量情节日志

### 解法：Summarizer + Gate

```
Episodic（活动日志）
        │
        │  不满阈值：不巩固（避免双写、浪费）
        ▼
   Gate（门控）
   例：每 20 轮对话 / 每 100 次活动
        │
        ▼
  Summarizer Agent
        │
        ▼
  Steal into Facts
        │
        ▼
  Semantic Memory（压缩后的核心事实）
```

要点：

1. **不要**每条新活动都立刻写进 Semantic（否则等于存两遍，失去「耐久事实」的意义）
2. 达到一定聊天 / 活动次数后再巩固
3. Summarizer 把情节压缩成「以后必须知道的事实」写入 Semantic
4. 效果：省 Token、更快、跨会话仍记得「你在做什么公司 / 是否 pivot」

---

## 七、Token 效率为什么成立

| 机制 | 作用 |
|------|------|
| RAG Top-K | 只取相关片段，不全量灌上下文 |
| Semantic 耐久事实 | 常用画像 / 业务事实短而稳，不必每次扫全量情节 |
| Gate + Summarizer | 控制巩固频率，避免重复存储与噪声膨胀 |
| Procedural 技能文件 | 行为规范外置，不靠海量对话「学出来」 |

一句话：**记全量情节，用摘要事实；按需召回，门控写入。**

## 八、电商客服例子（串起来）

| 层 | 内容 |
|----|------|
| System Prompt | 「你是本站客服 Bot」 |
| Chat History | 当前这一通会话 |
| Procedural | 生气客户 → 道歉、礼貌回复（`skill.md`） |
| Semantic | 产品说明、品牌话术、该客户画像 |
| Episodic | 历史订单、投诉、过往会话时间线 |
| 巩固 | 多轮后摘要：「该客户偏好 / 近期纠纷结论」→ 写入 Semantic |

## 九、面试口述版（可直接背）

Agent Memory 不是「把历史全塞进 Prompt」，而是一层上下文 + 三类记忆：

1. **Working Memory**：本轮 System Prompt + 会话历史 + 用户问题，会话结束即丢；
2. **Procedural**：技能 / 行为规范，文件化（如 skill.md）；
3. **Semantic**：耐久事实与画像，向量库 + RAG Top-K；
4. **Episodic**：带时间戳的活动日志，同样向量检索；
5. **Gate + Summarizer**：情节积累到阈值后，摘要「偷」进语义事实，避免全量扫日志、控制 Token。

现代产品（ChatGPT Memory 等）本质就是：情节记流水，语义记精华，工作内存只装当前该看见的那一截。
