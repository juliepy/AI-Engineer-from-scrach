# Agentic AI 智能体面试精简问答（面试口述版，好记、直接背诵）
## 1. 故障自动分诊智能体设计
Q：如何用智能体搭建企业故障告警自动处置流程？
A：
1. 接入监控数据源，接收云监控、DataDog告警事件；
2. 智能体对告警分类，判定故障等级和关联业务；
3. 调用工具检索历史同类故障、运维手册；
4. 通过API自动执行标准修复操作；
5. 全程记录操作日志，同步更新Jira/ServiceNow工单。
简单类比：智能体相当于急诊护士，判断症状、查阅过往病例、按规范处理，处理不了再升级。

## 2. 工具调用安全方案
Q：如何保证智能体调用外部工具的安全性？
A：
1. 工具调用做身份认证，分配最小范围权限；
2. 删除、改配置等高风险操作强制人工审批；
3. 全流程留存审计日志，每一步操作可追溯；
4. 提供操作回滚能力，出错可撤销。
类比逻辑：和网银一致，多重验证、交易留痕、支持转账撤回。

## 3. 多智能体系统内存的作用
Q：多智能体工作流中内存模块有什么核心价值？
A：
内存用来持久化整条流程上下文，保存历史决策、故障根因证据，持续维护流程状态。
没有内存，各个智能体无法互通历史信息，业务流程会断裂。
类比：就像电商订单号，串联下单、发货、售后全流程信息。

## 4. 可信可校验RAG流水线设计
Q：怎么构建一套检索结果可校验、高可靠的RAG系统？
A：
1. 做领域隔离，只检索企业内部专业知识库；
2. 生成内容附带文档来源引用；
3. 给检索结果设置置信度分数，过滤低可靠数据；
4. 完整检索链路可审计溯源。
类比：查医疗资讯只采信权威平台，多方核对信息再采纳。

## 5. 人在回路（人工介入）设计
Q：智能体自动化流程里，哪些场景必须加入人工审核？
A：
四类场景需要人工介入：无标准化方案的复杂异常、高风险变更操作、需要升级上报事件、金融/医疗等强合规监管业务。
类比：机场安检，机器自动扫描，可疑物品交由人工复核判断。

# 面试核心考察要点（收尾加分话术）
面试官重点考察：业务流程编排、多工具集成、生产环境风险管控、企业级落地架构思维，不是单纯写提示词。
面试官不关注：Transformer底层从零实现、LangGraph底层源码、复杂机器学习数学算法，只需基础了解即可。
